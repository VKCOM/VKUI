# Button [tag:component]

Компонент, который даёт пользователю возможность выполнить какое‑то действие или перейти на другую страницу.
Правильный `HTML`-тег определяется автоматически (по умолчанию `button` или `a` при передаче свойства `href`).

Связанные компоненты:

- [`ButtonGroup`](/components/button-group)

</Overview>

{/* @example-description: Кнопка с текстом. */}
<Playground>
  ```jsx
  <Button>Кнопка</Button>
  ```
</Playground>

## Режимы отображения

Задаётся свойством `mode`.

- `"primary"` — акцентная кнопка для основных действий (например, «Сохранить», «Отправить», «Открыть»);
- `"secondary"` — второстепенная кнопка для важных, но не основных действий (например, «Отмена», «Редактировать»). Часто используется в паре с `primary`;
- `"outline"` — второстепенная кнопка с обводкой и средним акцентом. Подходит, когда нужно сохранить визуальную лёгкость интерфейса, или когда фон `secondary` сливается с фоном интерфейса/баннера;
- `"tertiary"` — кнопка без фона с низким акцентом для действий с минимальным влиянием на интерфейс (например, «Помощь», «Подробнее», «Справка»). Применяется, когда `primary` и `secondary` уже заданы;
- `"link"` — визуально как `tertiary`, но без фона и боковых отступов. Удобна в шапках или в ряду с элементами интерфейса, где нужен небольшой отступ между объектами.

## Размеры

Задаётся свойством `size`.

Значения, соответствующие каждому размеру, зависят от параметра адаптивности `density`.
В режиме `density="compact"` значения каждого из размеров будут меньше, чем в режиме `density="regular"`.

{/* @example-description: Сравнение размеров `s/m/l` в режимах адаптивности `compact` и `regular`. */}
<Playground direction="column">
  ```jsx
  <AdaptivityProvider density="compact">
    <Flex gap="m" align="center">
      <Button size="s">s</Button>
      <Button size="m">m</Button>
      <Button size="l">l</Button>
    </Flex>
  </AdaptivityProvider>
  <AdaptivityProvider density="regular">
    <Flex gap="m" align="center">
      <Button size="s">s</Button>
      <Button size="m">m</Button>
      <Button size="l">l</Button>
    </Flex>
  </AdaptivityProvider>
  ```
</Playground>

## Состояния

### `disabled`

Отключает взаимодействие с кнопкой и добавляет визуальную индикацию недоступности.

{/* @example-description: Кнопка в состоянии `disabled` без возможности взаимодействия. */}
<Playground>
  ```jsx
  <Button disabled>Button</Button>
  ```
</Playground>

### `loading`

Показывает индикацию загрузки вместо содержимого кнопки. Используйте, когда после нажатия запускаются длительные действия.

⚠️ **Важно для доступности**: при `loading={true}` компонент автоматически устанавливает `aria-label` в значение `loadingLabel` (по умолчанию «Загрузка...»), чтобы скринридер объявил контекст загрузки. Значение можно переопределить через `loadingLabel`.

{/* @example-description: Состояние загрузки кнопки с дефолтным и кастомным `loadingLabel`. */}
<Playground>
  ```jsx
<Button loading>Сохранить</Button>
<Button loading loadingLabel="Сохранение данных...">Сохранить</Button>
  ```
</Playground>

## Контент в начале/в конце

Слева и/или справа от текста можно добавить контент через свойства `before` и `after` соответственно. Чаще всего это иконки.

Рекомендации по размеру иконок:

- `size="s"` — `12px`;
- `size="m"` — `16px`;
- `size="l"` — `24px`.

Для `size="l"` в `before`/`after` также подойдёт компонент `Counter`.

`before`/`after` можно использовать без `children` — получится кнопка только с иконкой. Такие кнопки требуют дополнительных действий для доступности — см. раздел [`a11y`](#a11y).

{/* @example-description: Добавление контента в `before/after`: иконки и счётчик в кнопках разных размеров. */}
<Playground>
  ```jsx
  <Button after={<Icon12Like />} size="s" aria-label="Добавить в избранное" />
  <Button before={<Icon16Add />} size="m">
    Добавить
  </Button>
  <Button after={<Counter>16</Counter>} size="l">
    Открыть
  </Button>
  ```
</Playground>

## Доступность (a11y) [#a11y]

Компонент автоматически выбирает правильный `HTML`-тег (`button` по умолчанию или `a` при передаче `href`), обеспечивая базовые требования доступности.

Поддерживаются все стандартные `aria`-атрибуты на случай, если нужно переопределить стандартное поведение.

При одновременной передаче `href` и `disabled` VKUI не передаёт `href` тегу `a`, отключая взаимодействие. По возможности не передавайте `href` и `disabled` вместе — это не соответствует `a11y`.

Для кнопки только с иконкой (`before`/`after` без `children`) обязательно добавьте `aria-label`. Ещё лучше — использовать `VisuallyHidden` с нужным текстом: это универсальнее для ассистивных технологий.

```jsx
// хорошо
<Button before={<Icon16Search />} size="m" aria-label="Найти" />

// ещё лучше
<Button before={<Icon16Search />} size="m">
  <VisuallyHidden>Найти</VisuallyHidden>
</Button>
```

### disableSpinnerAnimation

Свойство `disableSpinnerAnimation` отключает анимацию при `loading={true}`.
