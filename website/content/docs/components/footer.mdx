<Overview>

# Footer

Компонент для создания семантических подвалов. Обеспечивает правильную структуру документа и доступность.

</Overview>

## Доступность (a11y)

### 1. Семантическая роль

- По умолчанию использует HTML-тег `<footer>` с ролью `contentinfo`
- Предназначен для:
- Контактной информации
- Ссылок на разделы сайта
- Копирайта и юридических данных
- Мета-информации о содержимом

### 2. Ключевые правила

| Сценарий                   | Решение                       | Почему важно                      |
| -------------------------- | ----------------------------- | --------------------------------- |
| Глобальный подвал страницы | Только один на страницу       | Соответствие стандартам WAI-ARIA  |
| Несколько подвалов         | Добавить `aria-label` каждому | Различение разделов скринридерами |
| Декоративный элемент       | `<Footer Component="div">`    | Избежать ложной семантики         |

### 3. Контекст использования

```jsx static
// ✅ Правильно: внутри секционного контента
<Group> // ← Тег <section>
<Content>
<Footer>Копирайт 2024</Footer>
</Group>

// ❌ Опасный пример: глобальный подвал
<Panel>
<Content>
<Footer>Контакты</Footer> // ← Может считаться глобальным
</Panel>
```

### 4. Решение проблем

**Проблема**:
Несколько подвалов без контекста → скринридеры воспринимают как дубли

**Решение**:

```jsx static
// Добавляем явные метки
<Footer aria-label="Навигация">
<Link>О нас</Link>
<Link>Контакты</Link>
</Footer>

<Footer aria-label="Юридическая информация">
<Text>© 2024 Компания</Text>
</Footer>
```

### 5. Когда менять тег

Используйте `Component="div"` если:

- Контент не соответствует семантике подвала
- Элемент используется для визуального оформления
- Уже есть глобальный подвал на странице

```jsx static
// Декоративный раздел
<Footer Component="div" className="custom-style">
  <RatingStars value={4} />
</Footer>
```
