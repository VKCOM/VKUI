# @vkontakte/vkui-mcp

MCP-сервер для документации VKUI: даёт AI-ассистентам доступ к списку компонентов, их свойствам и примерам кода.

## Подключение через npx

Сервер можно запускать без установки пакета в проект:

```bash
npx @vkontakte/vkui-mcp
```

Или с явной версией:

```bash
npx @vkontakte/vkui-mcp@latest
```

## Настройка в Cursor / MCP-клиенте

В конфиг MCP (например `.cursor/mcp.json` или настройки Cursor) добавьте:

```json
{
  "mcpServers": {
    "vkui": {
      "command": "npx",
      "args": ["-y", "@vkontakte/vkui-mcp"]
    }
  }
}
```

Опция `-y` у `npx` разрешает автоматическую установку пакета при первом запуске.

## Переменные окружения

- **`VKUI_MCP_BASE_URL`** — базовый URL MCP-данных (по умолчанию `https://vkui-mcp.cdn-vk.ru`).
- **`VKUI_VERSION`** — версия VKUI в MCP-данных (по умолчанию `latest`).

Итоговый адрес данных формируется как `{VKUI_MCP_BASE_URL}/{VKUI_VERSION}`.

## Ресурсы (resources)

Клиент может подключать данные как контекст через чтение ресурсов (без вызова инструментов):

- **`vkui://migration/v8`** — статический ресурс: список всех целей миграции на v8 (JSON).
- **Шаблон `vkui://component/{slug}`** — карточка компонента по slug (описание и props). В списке ресурсов отображаются все компоненты; автодополнение по `slug`.
- **Шаблон `vkui://migration/{name}`** — рекомендации по миграции для одного компонента или хука по имени. В списке — все цели миграции; автодополнение по `name`.

## Инструменты (tools)

- `list_components` — список всех компонентов
- `get_component_metadata` — карточка компонента (описание, props)
- `list_hooks` — список всех хуков
- `get_hook_metadata` — карточка хука (описание, props)
- `get_examples` — примеры использования компонента или хука (по `name` или `slug`)
- `list_migration_targets` — список компонентов и хуков для миграции на v8
- `get_migration_target` — рекомендации по миграции конкретного компонента или хука

## Для разработчиков пакета

- **Сборка и тесты:** `yarn test` выполняет сборку и запускает тесты (unit-тесты провайдера миграций и интеграционные тесты через stdio с моком HTTP-документации).
- **Сборка:** в директории пакета выполните `yarn build` — в `dist/` попадут скомпилированные JS и типы.
- **Публикация:** при `npm publish` или `yarn npm publish` автоматически вызывается `prepack`, который запускает `yarn run build`. В npm-пакет включается только папка `dist/` (см. поле `files` в `package.json`), поэтому пользователям не нужен TypeScript — достаточно установить пакет и запустить `vkui-mcp` из bin.
