# /check-pr

Запускает полный набор проверок, аналогичный CI-пайплайну для PR. Позволяет обнаружить проблемы до пуша.

## Аргументы

- Без аргументов — запустить все проверки (lint + test + build)
- `--full` — запустить все проверки включая E2E
- `--quick` — запустить только lint (без тестов и билда)
- `--only <check>` — запустить конкретную проверку: `lint`, `types`, `test`, `build`, `e2e`

## Логика работы

### Шаг 0: Определить набор проверок

По аргументам определить что запускать:

| Аргумент | Проверки                             |
|----------|--------------------------------------|
| (без аргументов) | lint → test → build                  |
| `--quick` | lint (все линтеры)                   |
| `--full` | lint → test → build → e2e            |
| `--only lint` | lint:format -> lint:es -> lint:style |
| `--only types` | lint:types                           |
| `--only test` | test                                 |
| `--only build` | build:vkui                           |
| `--only e2e` | test:e2e                             |

### Шаг 1: Lint — проверка кода

Запустить `yarn lint`. Эта команда внутри запускает параллельно через `concurrently`:
- `lint:types` — TypeScript типы во всех воркспейсах
- `lint:es` — ESLint
- `lint:style` — Stylelint
- `lint:format` — Biome форматирование

**Если `lint` упал:**

- `lint:types` → показать ошибки типов, предложить исправления
- `lint:es` → предложить запустить `yarn lint:es --fix` для автофикса, затем исправить остальное вручную
- `lint:style` → предложить запустить `yarn lint:style --fix`, типичные проблемы: физические CSS-свойства вместо логических, порядок стилей
- `lint:format` → предложить запустить `yarn lint:format --fix`

**Не останавливаться на ошибках линтера** — продолжить остальные проверки, чтобы дать полную картину. Но сохранить статус ошибки.

### Шаг 2: Unit-тесты

Запустить `yarn test`.

- Если есть падения — показать первые 3 ошибки с файлами и номерами строк
- Сохранить статус

### Шаг 3: Build

Запустить `yarn build:vkui`.

Эта команда внутри:
1. Очищает `dist/`
2. Генерирует CSS custom medias и CSS modules types
3. Параллельно: `tsc` (декларации) + SWC (es6 + cssm) + PostCSS

Если билд упал — показать ошибку компиляции.

### Шаг 4: E2E-тесты (только при `--full` или `--only e2e`)

Запустить скилл `/e2e` без аргументов — он проверит все компоненты. Если нужно запустить E2E для конкретных компонентов, использовать `/e2e <ComponentName>`.

Предупредить пользователя что это долго (5-15 минут).

### Шаг 5: Итоговый отчёт

После всех проверок вывести сводку:

```
📋 Результат проверки PR:
  ✅ lint:format    — OK
  ✅ lint:style     — OK
  ❌ lint:types     — 3 ошибки
  ✅ lint:es        — OK
  ✅ lint:generated — OK
  ✅ test           — 1420/1420 прошло
  ✅ build          — OK
  ⏭️  e2e           — пропущено (запусти с --full)

❌ PR не готов к мёржу — есть 1 проблема.
```

Если все проверки прошли:

```
📋 Результат проверки PR:
  ✅ lint:format    — OK
  ✅ lint:style     — OK
  ✅ lint:types     — OK
  ✅ lint:es        — OK
  ✅ test           — 1420/1420 прошло
  ✅ build          — OK

✅ PR готов к мёржу!
```

## Важные замечания

- Запускать проверки последовательно: lint → test → build → e2e. Не запускать параллельно — некоторые проверки (build) очищают `dist/` и могут конфликтовать
- Если `yarn lint:types` падает из-за отсутствующего `dist/`, это нормально — `lint:types` проверяет исходники, а не билд
- E2E по умолчанию пропускается — слишком долго для локальной проверки
