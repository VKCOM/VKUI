# /run-tests

Запускает тесты для VKUI. Определяет что именно запустить по аргументам пользователя.

## Аргументы

- Без аргументов — запустить все unit-тесты (`node --run test`)
- Имя компонента (например, `Button` или `PanelHeader`) — запустить unit-тесты и E2E-тесты для этого компонента
- Путь к файлу — запустить тесты по указанному пути
- `--e2e` — запустить только E2E-тесты (для компонента или все)
- `--unit` — запустить только unit-тесты (для компонента или все)

## Логика

### Unit-тесты

1. **Все тесты:** `node --run test`
2. **Тесты компонента:** если передано имя компонента (например, `Button`):
   - Запустить: `yarn run -T vitest run <ComponentName> --pool=vmThreads`
3. **Тесты по пути:** `yarn run -T vitest run <путь> --pool=vmThreads`

### E2E-тесты

Если указан компонент — использовать скилл `/e2e <ComponentName>`. Он настроит `.env` и запустит E2E.

Если компонент не указан — использовать `/e2e` без аргументов (все E2E-тесты).

### Компонент с обоими типами тестов (по умолчанию)

Если передано имя компонента без флагов `--unit`/`--e2e`:

1. Запустить unit-тесты для компонента
2. Запустить E2E-тесты через `/e2e <ComponentName>`

## Примеры

```
/run-tests                          — все unit-тесты
/run-tests Button                   — unit + E2E для Button
/run-tests Button --unit            — только unit-тесты для Button
/run-tests Button --e2e             — только E2E для Button
/run-tests packages/vkui/src/components/Button/Button.test.tsx  — по пути
```

## Вывод

После запуска кратко сообщи результат: сколько тестов прошло/упало, если есть падения — покажи первые ошибки. Не дублируй весь вывод.
