# /lint-fix

Анализирует и исправляет ошибки линтеров и форматирования в VKUI.

## Логика работы

### Шаг 1: Запустить проверку

Запусти `yarn lint` чтобы получить полный отчёт по всем проверкам:
- `lint:types` — проверка типов TypeScript
- `lint:es` — ESLint
- `lint:style` — Stylelint
- `lint:format` — Biome форматирование

### Шаг 2: Автоматическое исправление

Попробуй исправить то, что можно автоматически:

1. **Biome форматирование:** `yarn lint:format --fix` (если есть ошибки форматирования)
2. **ESLint:** `yarn lint:es --fix` (если есть fixable ошибки)
3. **Stylelint:** `yarn lint:style --fix` (если есть fixable ошибки)

### Шаг 3: Анализ оставшихся ошибок

Если после автофикса остались ошибки, проанализируй их и предложи исправления:

- **Ошибки типов (`lint:types`):** найди файл с ошибкой и исправь тип. Частые причины:
  - Отсутствует экспорт типа
  - Несовпадение пропсов
  - Неправильный generic

- **Stylelint ошибки:** частые причины:
  - Использование физических CSS-свойств вместо логических (`margin-left` вместо `margin-inline-start`)
  - Неизвестные CSS custom properties (опечатка в имени токена)
  - Неправильный порядок стилей

- **ESLint ошибки:** частые причины:
  - Нарушение порядка импортов
  - Использование `any` типа
  - Missing `type` в type-only импортах (должно быть `import type { Foo } from '...'`)

### Шаг 4: Повторная проверка

После исправлений запусти `yarn lint` снова, чтобы убедиться что все ошибки устранены.

## Конфиги

При исправлении ошибок ориентируйся на конфиги проекта:
- **Форматирование:** `biome.json`
- **ESLint:** `.eslintrc.json`
- **Stylelint:** `@vkontakte/stylelint-config` + `stylelint-config-recess-order`
- **TypeScript:** `tsconfig.json` + `packages/vkui/tsconfig.dist.json`
